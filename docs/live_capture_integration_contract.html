Live Capture Integration Contract

Scope boundary: my_git_documents is the browser and UI layer only. The execution layer remains the separate local toolkit at C:\Users\andrew\PROJECTS\interview_capture_transcription_x1.

The browser must never control OBS Studio, PowerShell, microphone routing, or desktop capture directly. All orchestration flows through a localhost JSON bridge bound to 127.0.0.1:8765.

Ownership Model

Layer Repository / Path Responsibility
Web UI C:\Users\andrew\PROJECTS\GitHub\my_git_documents Start and stop requests, status polling, UI state, warnings, result links, history display, and interview prep context.
Local toolkit C:\Users\andrew\PROJECTS\interview_capture_transcription_x1 Recording orchestration, transcription, packaging, manifests, logs, and bridge-hosted artifact access.

Bridge Base Configuration

The repo-side integration centralizes the bridge base URL and endpoint templates in a workspace-level config. The UI must not scatter http://127.0.0.1:8765 through multiple pages or inline scripts.

{
  "bridgeBaseUrl": "http://127.0.0.1:8765",
  "artifactRouteTemplate": "/api/runs/{run_id}/artifact/{artifact_key}",
  "endpoints": {
    "captureStart": "/api/capture/start",
    "captureStop": "/api/capture/stop",
    "captureStatus": "/api/capture/status",
    "transcriptionStart": "/api/transcription/start",
    "runs": "/api/runs",
    "runDetailsTemplate": "/api/runs/{run_id}"
  }
}

Required Endpoints

  • POST /api/capture/start
  • POST /api/capture/stop
  • GET /api/capture/status
  • POST /api/transcription/start
  • GET /api/runs
  • GET /api/runs/{run_id}

The UI also supports a bridge-hosted artifact route when available so transcript and ZIP links remain browser-safe even when raw output values are local Windows paths.

The repo-side parser treats missing optional fields as empty values and preserves unknown future fields.

Browser-Safe Output Rules

  • If an output is already an HTTP or HTTPS URL, the page links to it directly.
  • If an output is a bridge-relative path, the page resolves it against the bridge base URL.
  • If an output is only a local Windows filesystem path, the page does not attempt direct browser file access.
  • When direct web linking is not available, the page falls back to the bridge artifact route template for transcript, ZIP, summary, log, and recording retrieval.

Transport vs Run Failure

Condition UI handling
Bridge unreachable, CORS failure, timeout, malformed JSON, non-2xx transport issue Show local capture service offline, set service badge to offline, keep prior run details visible when possible, and retry polling on the offline cadence.
Bridge reachable but run status is warn or fail Keep the bridge online, render run state as returned, surface warnings, and allow the user to inspect logs and outputs.
