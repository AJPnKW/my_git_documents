Opportunity Design

Role-specific opportunity layer for Venuiti / DevStaff Canada / Venuiti Healthcare, centered on the Technology Project Manager role in Waterloo, Ontario.

Design Intent

  • Keep the opportunity layer specific to this role instead of generating a reusable master artifact.
  • Translate the posting into interview-ready priorities, story selection rules, and answer controls.
  • Bias the package toward delivery rigor, technical-team fluency, healthcare adjacency, and regulated-environment credibility.

Opportunity Shape

  • Target company: Venuiti
  • Target brand: Venuiti Healthcare
  • Role title: Technology Project Manager
  • Work model: 100 percent in-office in Waterloo
  • Core ask: full-lifecycle software delivery with scope, budget, timeline, documentation, and stakeholder control
  • Role pressure points: healthcare context, security-clearance requirement, external-partner coordination, and visible reporting discipline

Story Strategy

  • Primary stories: Sun Life health and benefits delivery, Manulife outsourced-project recovery, Manulife internet-release delivery
  • Secondary stories: Spaenaur application portfolio leadership, Hubble zero-downtime ERP upgrade, Kuntz HRMS modernization
  • Selective story: OpenText security governance, used when the interviewer leans into compliance, security, or documentation rigor
  • Reasoning: the best answers need domain adjacency plus strong execution evidence, not a generic PM narrative

Risk Controls

  • Do not overstate direct healthcare depth.
  • Do not let recent cybersecurity work displace software-delivery credibility.
  • Answer on-site and citizenship questions directly because they appear to be screening items.
  • Use concrete projects and outcomes before discussing frameworks or certifications.

Validation

  • Required filenames use opportunity_profile.json and opportunity_interview_response_bank.json, not job_posting.json.
  • All generated documentation for this task is HTML.
  • The opportunity package is deterministic at the content level and uses fixed role identifiers.
  • Missing source inputs are recorded explicitly so downstream users know which sections are inferred from available materials.
