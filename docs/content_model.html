Content Model

The repo now treats files/ as the primary content root. Folder structure defines the navigation tree, normalized filenames define document roles, and generated workspace JSON feeds the shared viewer shell.

  • files/single_page/<topic>/ is for a single primary document workspace such as Yellow House.
  • files/multi_page/<topic>/ is reserved for future ordered multi-document workspaces.
  • files/resumes/<company>/<posting>/ holds normalized interview artifacts.
  • Normalized resume artifact names currently include posting.html, resume.html, posting_analysis.html, interview_prep.html, and interview_notes.html.
  • app/data/workspaces/ and web/data/workspaces/ are generated from the files/ tree by scripts/build_site_index.py.
