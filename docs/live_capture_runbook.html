Live Capture Runbook

This runbook covers the repo-side workflow for the live interview capture integration. It assumes the separate toolkit at C:\Users\andrew\PROJECTS\interview_capture_transcription_x1 is installed and operating independently.

Normal Session Flow

  1. Start the local toolkit and its localhost bridge so it listens on 127.0.0.1:8765.
  2. Open the Venuiti interview mode page in my_git_documents and launch the dedicated live-capture workspace.
  3. Confirm the panel no longer shows local capture service offline.
  4. Review the prep context loaded from the workspace data sources.
  5. Press Start Capture to create a new run.
  6. Use Stop Capture when the session ends.
  7. If the backend does not auto-transcribe on stop, press Start Transcription once the run reaches recorded.
  8. Monitor status transitions through transcribing, packaging, and complete.
  9. Open transcript, summary, log, or ZIP links once they become ready.

What The Browser Must Not Do

  • Do not embed PowerShell commands in webpage code.
  • Do not attempt direct OBS control from browser JavaScript.
  • Do not access raw Windows paths with file:// links as the primary result flow.
  • Do not copy the local toolkit into this repository.

Offline Recovery

  • If the panel shows local capture service offline, verify that the localhost bridge is running and reachable.
  • The UI retries automatically on the offline polling cadence, so a page refresh is not required once the bridge comes back.
  • If the bridge recovers mid-session, the panel reloads the current run from GET /api/capture/status.

Expected Output Handling

  • The panel treats transcript text, transcript SRT, summary JSON, log file, recording file, and ZIP bundle as bridge-owned outputs.
  • Browser-safe URLs are preferred. Artifact-route fallback is used when the bridge exposes files that way.
  • Warnings are visible but non-blocking, so the user can still inspect partial results.
