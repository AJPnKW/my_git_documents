Security Model

The current baseline uses neutral access-entry screens for local/private use. This is a usability gate, not strong hosted protection. Real hosted privacy would require server-side auth or a protected hosting layer.

  • No live secrets are committed in tracked config.
  • Local-only auth files are ignored and should not be packaged.
  • Yellow House and Resumes use separate gate keys and can evolve independently.
